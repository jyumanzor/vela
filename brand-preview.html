  • Copy starter CLAUDE.md template
  • Set domain (white papers → HBR)
  • Load relevant failure modes
  • Define output contracts

Operating Loop

Methodology
Build → Challenge → Resolve → Promote. Every non-trivial deliverable runs through all four steps. The challenger can be a second model, an adversarial prompt, or your own checklist. The point is: nothing ships unchallenged.

Plausible-but-Wrong Numbers

Failure Mode
AI generates statistics that feel correct but were never verified — magnitude errors, wrong dates, fabricated percentages. Every cited number needs a provenance tag: PUBLISHED, DERIVED, or ANALYST.

Cross-Model Verification

Verification
Before submitting anything with cited claims, run your argument through an adversarial model. The second model's job is to find what you missed. Three protocols: adversarial review, parallel generation, acceptance testing.

Build Log Protocol

Methodology
One append-only file per project: _BUILD_LOG.md. What was built, what's canonical, what changed, what broke, what was learned. Read it before starting. Write it before stopping. That's the whole system.
